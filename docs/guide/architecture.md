# 系统架构

> 深入了解 Kunnix 的内部设计

## 🏗️ 整体架构

Kunnix 采用模块化设计，包含三大核心系统：

```
┌─────────────────────────────────────┐
│    Kunnix Memory & Task System      │
├─────────────────────────────────────┤
│                                     │
│  ┌──────────┐  ┌──────────┐        │
│  │ Memory   │  │  Task    │        │
│  │ System   │  │Orchestrator       │
│  └──────────┘  └──────────        │
│         │              │            │
│         └──────┬───────┘            │
│                │                    │
│         ┌──────▼───────┐            │
│         │SkillEvolution│            │
│         └──────────────┘            │
└─────────────────────────────────────┘
```

## 📊 核心模块

### 1. 记忆系统

- 混合检索（全文 + 向量）
- L0-L5 分级架构
- RRF 融合 + MMR 重排

### 2. 任务管理

- 自动提取
- 状态追踪
- 记忆关联

### 3. 技能进化

- 模式识别
- 技能生成
- 审核发布

## 🎯 设计原则

1. **模块化** - 职责单一
2. **可扩展** - 易于添加功能
3. **高性能** - 响应<500ms
4. **易用性** - 简洁 API
